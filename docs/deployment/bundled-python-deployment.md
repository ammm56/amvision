# 同目录 Python 部署与发布方案

## 文档目的

本文档用于把“发布阶段使用项目同目录 Python 运行时”的约束落实为可执行部署方案，覆盖发布包结构、安装步骤、升级回滚和验证方式。

本文档不依赖目标机器预装系统 Python，也不假设现场额外具备 conda 环境。

## 适用范围

- standalone、workstation、edge 三类发布形态
- 同目录 Python 运行时的目录布局和启动约定
- 发布包安装、升级、回滚和排障流程
- 需要单独交付的系统依赖边界

## 发布包交付要求

每个发布包至少应包含以下内容：

- python：同目录 Python 解释器与依赖
- app：后端服务和 worker 代码
- frontend：前端静态资源
- plugins：基础插件目录和可选插件位置
- config：默认配置模板与环境配置文件
- data：本地对象存储、缓存或运行期数据目录
- logs：服务与 worker 日志目录
- manifests：版本、兼容性、依赖和构建信息
- launchers：启动、停止、健康检查、版本显示和回滚辅助脚本

## 目标机前置条件

- 满足目标操作系统版本要求
- 若使用 GPU 或特定推理后端，已安装对应驱动或厂商运行时
- 具备所需端口、文件权限和磁盘空间
- 如需对接外部系统，已准备相应网络或证书条件

## 安装流程

### 1. 解压或放置发布包

- 将完整发布包放到目标目录
- 确保应用目录、数据目录和日志目录具备读写权限

### 2. 初始化配置

- 复制默认配置模板为现场配置文件
- 配置数据库路径、对象存储路径、端口、外部系统端点和插件启用项
- 如果采用 side-by-side 升级策略，应确保 config 与 data 可独立保留

### 3. 执行环境检查

- 运行维护脚本或等价入口检查 bundled Python 是否可启动
- 检查 runtime manifest、插件目录和必要依赖是否完整
- 检查 GPU、ZeroMQ、本地端口和目录权限是否满足要求

### 4. 启动 backend-service

- 通过发布包自带启动器启动服务进程
- 启动器必须显式调用同目录 Python，而不是依赖 PATH 中的解释器
- 启动后检查健康接口、日志输出和静态资源访问

### 5. 启动 worker

- 按部署形态启动训练、推理、转换或流程 worker
- 验证 worker 能成功连接 QueueBackend、读取插件和写回状态

### 6. 执行冒烟验证

- 打开浏览器前端界面并确认基础页面可访问
- 调用 REST 健康检查与版本接口
- 建立 WebSocket 订阅并确认状态事件可达
- 提交一个最小任务，确认任务能排队、执行并写回结果

## 形态差异说明

### standalone

- 默认在单机内同时部署服务、worker、前端资源和本地数据目录
- 可启用本地 ZeroMQ 提供内部 IPC
- 适合功能完整但单机运行的现场环境

### workstation

- 强调工作站界面、局域网接入和外部系统协作
- 可按现场需要附带协议适配插件、模块连接插件或硬件桥接插件
- 应重点验证端口开放、前端可用性和集成端点回调路径

### edge

- 强调最小依赖集和资源约束
- 应裁剪不必要插件与前端资源
- 应重点验证运行时体积、内存占用和厂商运行时兼容性

## 升级流程

### 推荐策略

- 使用版本目录并行存放新旧发布包
- 将 config、data、logs 尽量独立于应用版本目录
- 通过切换启动器或服务指向的新版本目录完成升级

### 升级步骤

1. 备份当前 config、data 和关键日志
2. 解压新版本到独立目录
3. 执行环境检查与兼容性检查
4. 必要时执行数据库迁移或插件兼容性校验
5. 启动新版本服务和 worker
6. 完成冒烟验证后再切换正式流量或现场入口

## 回滚流程

1. 停止当前版本服务与 worker
2. 切回上一个完整版本目录
3. 恢复与该版本匹配的配置和插件启用状态
4. 重新执行健康检查和最小任务验证

## 需要单独列明的系统依赖

- GPU 驱动
- 厂商推理运行时
- 操作系统级通信依赖
- 客户现场网络、证书和权限配置

## 安装后验收清单

- bundled Python 路径正确且不依赖系统 Python
- REST 健康检查正常
- WebSocket 事件订阅正常
- worker 能消费队列并回写状态
- 插件目录可扫描，插件启停符合配置
- 前端静态资源可访问，基础页面正常
- 如启用 ZeroMQ，本地 IPC 信道正常

## 排障优先顺序

1. 先检查 bundled Python 是否被正确调用
2. 再检查配置文件、目录权限和依赖 manifest
3. 再检查队列连接、插件兼容性和状态回写链路
4. 最后检查外部系统、GPU 或厂商运行时等环境问题
