# ADR-0002 同目录 Python 发布运行时

## 背景

项目目标场景以本地部署、工作站和边缘设备为主，目标机器通常不适合再额外配置系统 Python、conda 或复杂的 Python 依赖环境。

同时，开发阶段仍需要一个可迭代、可复现的环境来支持测试和依赖更新。

## 决策

开发阶段使用 conda 管理 Python 环境；发布和部署阶段使用项目同目录 bundled Python 运行时。

- 服务、worker、CLI 和维护脚本统一由 bundled Python 启动
- 发布包尽量自带应用级依赖，不要求目标机器预装系统 Python
- 仅将 GPU 驱动、厂商运行时或操作系统级依赖作为单独前置条件列出

## 备选方案

### 依赖系统 Python

- 优点：发布包体积较小
- 缺点：目标机器环境不可控，依赖冲突和版本漂移风险高

### 发布阶段也依赖 conda

- 优点：开发与发布环境一致性较高
- 缺点：现场部署门槛高，不适合作为最终交付要求

## 影响

- 提高了发布包自包含程度和现场部署可控性
- 需要维护 bundled Python 结构、依赖 manifest、升级回滚与健康检查工具
- 需要单独管理系统级依赖与硬件兼容性说明

## 后续动作

- 维护 runtime-packaging 与 bundled-python-deployment 文档
- 为发布包提供统一启动器、维护脚本和验收清单
